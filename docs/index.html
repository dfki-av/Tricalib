TriCalib

An Interactive Target-Free and Keypoint-Based Extrinsic Calibration Tool for Tri-Modal Sensor Suites — developed at DFKI (German Research Center for Artificial Intelligence).

Python 3.10+ CC BY-NC-SA 4.0 PyQt6 · OpenCV · Open3D · PyVista

TriCalib lets you manually select corresponding 2D pixel coordinates on camera images and 3D points on a LiDAR point cloud, then solves for the extrinsic transformation matrices that align all sensors. It supports three sensor pairs:

📷 ↔ 🟢

RGB ↔ LiDAR

PnP-based extrinsic estimation from manually selected 2D–3D correspondences.

📷 ↔ ⚡

RGB ↔ Event

Essential matrix + pose recovery from matched pixels in both cameras.

⚡ ↔ 🟢

Event ↔ LiDAR

PnP from event camera 2D points and corresponding LiDAR 3D points.

🔗

Joint Optimization

Levenberg–Marquardt minimization of combined reprojection error across all three pairs simultaneously.

Installation

This repository uses Git LFS for large binary assets (point clouds and images in examples/).

# Install Git LFS (once per machine)
git lfs install

# Clone the repository
git clone git@github.com:dfki-av/Tricalib.git
cd tricalib

# Pull LFS objects
git lfs pull

# Install in editable mode
pip install -e .

Quick Start

python -m tricalib

Step-by-step Workflow

  1. Load your data

    Use File → Load RGB Image, Load Point Cloud, and Load Event Image to open your sensor data. Load camera intrinsics via File → Load Intrinsics (JSON with fx, fy, cx, cy for both cameras).

  2. Select RGB correspondences

    Right-click on the main image viewer to mark 2D points on the RGB image. Each point is drawn with a numbered yellow circle.

  3. Select LiDAR correspondences

    The PyVista 3D viewer opens automatically. Use the PyVista point-picker (right-click or the picker button) to select the matching 3D points on the point cloud.

  4. Select Event camera correspondences

    The event image viewer opens in a separate window. Right-click to select points that correspond to the same physical features.

  5. Compute calibration

    Select one of the Calibration menu options: pairwise PnP for individual sensor pairs, or Compute All for joint optimization across all three modalities.

  6. Verify with projection

    Use Calibration → Project … actions to open projection viewers. Check alignment visually and view pixel-level reprojection errors via the metrics toolbar button.

  7. Save your results

    Save calibration to JSON with File → Save Calibration. Optionally save selected points (Save Points) and the full session state (Save State) for later resumption.

Point Selection

💡

Select at least 6 well-distributed correspondences for reliable PnP and essential matrix estimation. Points near image borders and at varying depths improve geometric conditioning.

Points must be selected in the same order across all viewers — the n-th RGB point, the n-th LiDAR point, and the n-th Event point must all correspond to the same physical feature in the scene.

Undo

Click the Undo button in the left toolbar (↩ icon) to remove the last selected point simultaneously from all three viewers. This keeps the correspondence lists in sync.

Calibration Methods

Pairwise PnP (LiDAR ↔ RGB / LiDAR ↔ Event)

Uses OpenCV's cv2.solvePnP on the selected 2D–3D correspondences to recover the rotation and translation that maps LiDAR coordinates to the camera plane. The optional Stereo Rectification and Auto Axis Alignment toggles pre-condition the data before solving.

Essential Matrix (RGB ↔ Event)

Uses RANSAC-based cv2.findEssentialMat on normalized pixel coordinates from both cameras, followed by cv2.recoverPose to extract the relative rotation and translation.

Joint Optimization (Compute All)

Jointly optimizes all three transformations using scipy.optimize.least_squares (Levenberg–Marquardt method). The cost function is the sum of reprojection residuals across all three sensor pairs. Transformations are parameterized as unit quaternions + translation vectors (21 parameters total).

Interface Panels

Two docked panels are available on the right side of the main window.

Selected Points Table

Displays all currently selected correspondences in a table with eight columns:

ColumnDescription
#Correspondence index
RGB u / RGB vPixel coordinates in the RGB image (integers)
LiDAR X / Y / Z3D coordinates in the LiDAR frame (4 decimal places)
EV u / EV vPixel coordinates in the event camera image (integers)

Click Edit Points to toggle edit mode. In edit mode, double-click any cell to correct a value. Changes immediately update the internal point lists.

Calibration Results Panel

Shows the most recently computed extrinsic transform. The rotation matrix R and translation vector t are displayed in a formatted layout. Use the Copy to Clipboard button to export the values as plain text.

Tips & Troubleshooting

Point ordering matters

The n-th point in each modality list must correspond to the same physical feature. Use the Undo toolbar button if you accidentally select points out of order.

At least 6 points for stable calibration

While PnP requires only 4 points, 6–10 well-distributed correspondences significantly improve accuracy. Aim for points at different depths and spread across the image plane.

Use Joint Optimization when all three modalities are available

Compute All minimizes the combined reprojection error across all three pairs, producing more consistent results than sequential pairwise calibration.

Auto Axis Alignment for LiDAR

LiDAR sensors often use a Z-up coordinate convention while cameras use Z-forward. Enable Auto Axis Alignment before computing if your point cloud appears rotated 90° in the projection viewer.

⚠️

Disable Auto Axis Alignment (or load a calibration) before loading a pre-computed calibration JSON — the toggle is automatically disabled when a calibration is loaded to avoid double-applying the basis transform.

Video generation

The projection viewers can generate videos from image and point cloud sequences. Both directories must contain files with matching names (different extensions). Sort order is alphabetical, so use zero-padded frame indices in filenames.

Reinitialize vs Restart

  • Restart — clean slate, all data is cleared.
  • Reinitialize — saves the current session state to state_dict.json and restarts, automatically reloading all files.

PyVista plotter does not open on Linux

Ensure a display server (X11 or Wayland with XWayland) is available. Run with DISPLAY=:0 if needed. PyVista is intentionally run in a separate process to avoid Qt/OpenGL context conflicts.
